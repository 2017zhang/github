一、固定宽高，子元素绝对定位，top、left:50%,margin是负值（元素的宽高的一半）。

    
         .box_a {
            position: absolute;;
            top: 50%;
            left: 50%;
            margin-left: -50px;
            margin-top: -50px;
        }
123123

兼容性也很好，缺点是需要知道子元素的宽高

二、固定宽高，子元素绝对定位，各个方向的距离都是0，margin设为auto。

    
        .wp {
            position: relative;
        }
        .box_b {
            position: absolute;;
            top: 0;
            left: 0;
            right: 0;
            bottom: 0;
            margin: auto;
        }
123123

兼容性也很好，缺点是需要知道子元素的宽高

三、子元素不需要固定宽高，子元素绝对定位，transform的translate属性宽高设置为自身的-50%。

    
    .wp {
        position: relative;
    }
    .box_c {
        position: absolute;
        top: 50%;
        left: 50%;
        transform: translate(-50%, -50%);
    }
123123

Internet Explorer 9 需要前缀 -ms-。

四、利用行内元素居中属性也可以做到水平垂直居中


    .wp_b {
    border: 1px solid red;
    width: 300px;
    height: 300px;
    line-height: 300px;
    text-align: center;
    font-size: 0px;
    }
    .box_d {
    font-size: 16px;
    display: inline;

    }
123123

五、居中元素的宽高必须固定，使用绝对定位和计算属性


 .box_f {
        position: absolute;;
        top: calc(50% - 50px);
        left: calc(50% - 50px);
    }
123123

这种方法兼容性依赖calc的兼容性，缺点是需要知道子元素的宽高

六、曾经table被用来做页面布局，也能够实现水平垂直居中，但是会增加很多冗余代码


    .wp_c {
    text-align: center;
    }
    .box_e {
    display: inline-block;
    }
123123

abel单元格中的内容天然就是垂直居中的，只要添加一个水平居中属性就好了.

缺点是这种方法就是代码太冗余，而且也不是table的正确用法

七、使用css新增的table属性，把普通元素，变为table元素的现实效果，通过这个特性可以实现水平垂直居中


  .wp_g {
        display: table-cell;
        text-align: center;
        vertical-align: middle;
    }
    .box_g {
        display: inline-block;
    }
123123

这种方法和table一样的原理，但却没有那么多冗余代码，兼容性好

八、flex作为现代的布局方案，只需几行代码就可以优雅的做到水平垂直居中


  .wp_h {
        display: flex;
        justify-content: center;
        align-items: center;
    }
123123

目前在移动端已经完全可以使用flex了，PC端老版本会有兼容性情况

九、grid作为现代的布局方案，只需几行代码就可以优雅的做到水平垂直居中


    .wp_r {
    display: grid;
    }

    .box_r {
    align-self: center;
    justify-self: center;
    }
123123

代码量也很少，但兼容性不如flex，不推荐使用

总结

  • PC端有兼容性要求，宽高固定，推荐absolute + 负margin
  • PC端有兼容要求，宽高不固定，推荐css-table
  • PC端无兼容性要求，推荐flex
  • 移动端推荐使用flex
